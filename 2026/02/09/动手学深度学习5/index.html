一、本章定位

第五章的主题是 深度学习计算。前面几章已经学习了线性回归、softmax 回归和多层感知机，也初步掌握了“数据—模型—损失函数—优化器—训练循环”的基本流程。

但是，如果只会直接写一个 nn.Sequential 模型，还不够真正理解深度学习工程。实际项目中，模型往往具有更复杂的结构，例如：

  • 多个网络模块需要重复组合；
  • 不同层的参数需要单独访问和初始化；
  • 需要自己定义新的层；
  • 训练好的模型需要保存和加载；
  • 模型需要在 CPU 和 GPU 之间切换；
  • 大模型训练还要考虑设备、显存和计算效率。

因此，第五章的重点不是提出新的模型，而是讲清楚深度学习框架中几个非常关键的工程组件。

本章可以概括为一句话：

学会把神经网络看作可组合、可管理、可保存、可迁移到不同设备上的计算模块。

二、层和块：神经网络的基本组织方式

1. 从层到块

这里包含两个线性层和一个 ReLU 激活层。

但是在更复杂的模型中，网络不只是简单地一层接一层。例如 ResNet 中有残差块，Transformer 中有注意力模块，卷积神经网络中有卷积块和池化块。

因此，深度学习框架通常使用“块”来组织模型。

块可以理解为一个可调用的神经网络模块。它既可以是单个层，也可以是由多个层组合成的复杂结构。

2. 块的核心功能

一个块通常具备以下功能：

  1. 接收输入数据；
  2. 执行前向传播；
  3. 存储可学习参数；
  4. 支持反向传播；
  5. 可以嵌套其他块；
  6. 可以保存和加载参数。

也就是说，块不仅仅是一个函数，它还是一个带状态的计算单元。

五、参数初始化

参数初始化会影响模型训练的稳定性和收敛速度。

1. 默认初始化

如果不手动指定初始化方式，框架会使用默认初始化策略。对于简单模型，这通常可以正常工作。

但在深层网络中，合理的初始化非常重要。初始化不当可能导致：

  • 梯度消失；
  • 梯度爆炸；
  • 收敛速度慢；
  • 不同神经元学习到高度相似的特征。

七、延后初始化

1. 什么是延后初始化

延后初始化指的是：模型定义时，部分参数的形状还不能完全确定，框架会等到第一次看到输入数据时，再根据输入形状真正创建参数。

例如：

这里没有显式指定输入维度。框架会在第一次执行前向传播时，根据输入自动推断维度。

2. 为什么需要延后初始化

延后初始化的好处是代码更灵活。尤其是在复杂网络中，有些层的输入维度不容易手动计算。

例如卷积网络经过多次卷积、池化之后，特征图尺寸可能变化复杂。如果使用延后初始化，就可以减少手动计算维度的麻烦。

3. 延后初始化的注意点

延后初始化虽然方便，但也有需要注意的地方：

  1. 在第一次前向传播之前，参数可能还没有真正创建；
  2. 如果需要手动初始化参数，最好确认参数已经被初始化；
  3. 调试模型结构时，需要先用一批样本跑一次前向传播。

简单理解：延后初始化是“先搭框架，等数据来了再确定细节”。

十二、容易混淆的地方

1. 层和块不是完全不同的东西

层可以看作简单的块，块也可以由多个层组成。深度学习框架用统一的模块机制来管理它们。

2. nn.Sequential 不适合所有模型

如果模型只是从前到后顺序执行，Sequential 很方便；如果模型有分支、循环、跳跃连接或多个输入输出，就应该自定义 nn.Module。

3. 参数值和梯度不是一回事

param.data 表示参数本身的数值，param.grad 表示反向传播计算得到的梯度。

4. 保存模型时通常保存 state_dict

保存 state_dict 比保存整个模型对象更灵活，也更适合实际项目。

5. GPU 不会自动接管所有计算

必须明确把模型和数据都移动到 GPU 上。否则可能仍然在 CPU 上运行，或者因为设备不一致而报错。

十三、学习体会

第五章让我意识到，深度学习不只是“写公式”和“调参数”，还需要理解框架如何组织计算。

前面章节中，我们更多关注模型本身，比如线性回归、多层感知机和损失函数。而第五章更像是在讲深度学习工程的基础设施：模型如何封装成块，参数如何管理，模型如何保存，自定义层如何写，GPU 如何使用。
