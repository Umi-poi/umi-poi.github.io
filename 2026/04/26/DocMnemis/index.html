DocMnemis

作者: umi
发表于2026-04-26
更新于2026-04-26
深度学习

结合Opendataloader pdf、Mnemis、VISA

Minemis

Mnemis 原本处理的是长期对话记忆，它把历史消息组织成 Base Graph + Hierarchical Graph（双层图），再通过 System-1 相似度检索 和 System-2 全局层级选择 两条路线联合召回信息（双路检索）

Opendataloader pdf

输入pdf,输出 Markdown、JSON 和 HTML；JSON 中包含页面编号和 bounding box 坐标；支持阅读顺序、表格、标题层级、图片坐标、OCR、公式、图表描述等能力；它也明确强调这些能力面向 RAG，尤其是结构保留和坐标引用

VISA

让模型在检索到的页面截图中标出支持答案的证据区域，而不是只给文档级来源

DocMnemis

把Minemis尝试用于文档检索

创新点

  • 创新点一：PDF-native Evidence Graph
    不是把 PDF 粗暴切成 chunk，而是利用 OpenDataLoader PDF 的结构化元素和 bbox，把 PDF 转成带版面、阅读顺序和来源坐标的证据图。

  • 创新点二：Dual Hierarchical Graph
    同时构建：
    文档原生层级：章节、页面、标题、段落
    语义抽象层级：方法、实验、数据集、指标、结论
    前者保证结构一致性，后者保证语义覆盖性。

  • 创新点三：Dual-Route Retrieval
    同时使用：
    System-1：快速相似检索
    System-2：层级全局选择
    前者负责快，后者负责全。

  • 创新点四：Evidence Chain with Bounding-box Attribution
    不是简单引用第几页，而是输出：
    文档名
    页码
    元素类型
    元素编号
    bbox 坐标
    证据链
    从而提高来源可解释性。

流程

① opendataloader解析

OpenDataLoader PDF 的 schema 中，顶层内容元素存放在 kids 数组中；每个基础元素包含 type、page number、bounding box 等字段，且 bounding box 坐标格式是 [left, bottom, right, top]。
它支持的内容元素包括 paragraph、heading、caption、table、text block、list、image、header/footer 等，这些正好可以映射成 DocMnemis 的节点。

② 结构化证据单元构建层

OpenDataLoader PDF 输出的是解析结果，但还不是 RAG 的“证据单元”。所以第二步要把 JSON element 转成统一格式的 Evidence Unit。

⑤ 多索引构建层

图构建好后，不能只存图，还要建索引。
DocMnemis 至少需要五类索引。

5.1 关键词索引

利用BM25
处理精确术语、公式编号、图表编号、模型名、数据集名、指标名。

5.2 向量索引

Dense Vector
处理语义相似问题。

5.3 视觉索引

为每页截图建立视觉 embedding
处理图表、版面和截图类问题。

5.4 Entity / Claim 索引

支持事实检索和多跳推理。

5.5 来源索引

生成答案后回溯来源。

⑦ Dual-Route Retrieval 双路检索层

7.1 System 1

快速找到与问题最相似的页面、块、实体和论断
容易漏掉语义不完全相似但结构上相关的证据
不擅长全文枚举
不擅长从多个章节系统性找全答案

目标模态 target_modality

├── Dense 检索
├── Entity 检索
├── Claim 检索
└── Visual 检索，可选

初步候选

System-1 的结果可以作为提示，让 System-2 更快定位相关层级。

自上而下选择流程

以问题为例：“这篇论文有哪些实验结果证明方法有效？”

第一步选择一级类别：
方法设计
实验评估 ← 选中
相关工作
局限性

第二步进入“实验评估”：
数据集
评价指标
主实验结果 ← 选中
消融实验 ← 选中
案例分析 ← 选中
错误分析

第三步进入对应章节：
Section 4.1 Datasets
Section 4.2 Main Results
Section 4.3 Ablation Study
Section 4.4 Case Study

第四步选择证据节点：
Table 1
Table 2
Paragraph 4.2-3
Figure 5
Caption 5

最终返回：
System-2 candidates:
T1, T2, B4.2-3, F5, Cap5

为了避免遍历过多，设置停止条件：
达到最大层深
候选证据数量达到上限
节点相关性低于阈值
连续两层没有新增高分证据
token 预算不足

双路合并

System-1 和 System-2 会返回不同候选。
合并后形成候选证据池然后统一打分。

⑧ Evidence Chain 证据链重排层

Evidence Subgraph

候选证据不能直接交给大模型。要先做局部扩展，形成 Evidence Subgraph
例如检索到一个实验表格 T2，系统要自动扩展：

Evidence Chain

这是 DocMnemis 相比普通 RAG 的重要差异。
普通 RAG 是：Top-k chunks → LLM

DocMnemis 是：Top-k evidence → Graph expansion → Evidence chain → LLM

“本文使用了什么数据集？”
证据链：Question → Entity: dataset → Evidence Block→ Answer

⑨ Source-grounded Generation 带来源答案生成层

生成阶段必须限制大模型：
只能基于 Evidence Chain 回答；
每个关键结论都要绑定 evidence_id；
没有证据就回答“文档中未找到明确依据”；
输出页码、元素类型和 bbox。
