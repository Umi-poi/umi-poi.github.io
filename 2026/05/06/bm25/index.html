BM25：关键词检索中的经典相关性算法

作者: umi
发表于2026-05-06
更新于2026-05-06
RAG信息检索搜索算法

BM25

BM25（Best Matching 25）是信息检索里非常经典的排序算法。它常用于搜索引擎、站内搜索、文档检索以及 RAG 系统中的召回阶段。虽然现在向量检索和语义检索很流行，但 BM25 依然很重要，因为它对精确关键词匹配非常敏感，速度快、解释性强、实现简单，而且在很多真实业务场景里效果稳定。

一句话概括：

BM25 会根据查询词在文档中出现得多不多、这个词本身稀不稀有、文档是不是过长，来计算文档和查询的相关性分数。

为什么需要 BM25

最朴素的关键词检索方法是统计词频：查询词在文档中出现次数越多，文档越相关。

比如查询：

如果一篇文章里多次出现 vector 和 database，它看起来就应该更相关。

但只看词频会有几个问题：

  • 常见词不应该贡献太高分数，比如 the、is、的、是。
  • 一个词出现 100 次，不一定比出现 10 次相关 10 倍。
  • 长文档天然更容易包含查询词，容易拿到不公平的高分。
  • 不同查询词的重要性不同，稀有词通常更能区分文档。

BM25 的设计就是为了处理这些问题。它不是简单地数词，而是把词频、逆文档频率和文档长度归一化组合起来。

IDF：稀有词更重要

IDF 的意思是 Inverse Document Frequency，也就是逆文档频率。

如果一个词出现在几乎所有文档里，它区分文档的能力就很弱；如果一个词只出现在少数文档里，它就更有信息量。

BM25 和传统 TF-IDF 的一个重要区别是：BM25 对词频做了饱和处理。

  • 文档 B 出现 5 次。
  • 文档 C 出现 50 次。

长文档天然更容易命中查询词。如果不做处理，一篇很长的文章可能只是因为内容多，就比短而精准的文章得分更高。

有三篇文档：

  • 文档 A：短文，精准讲机器学习中的检索方法。
  • 文档 B：长文，包含机器学习、数据库、模型训练、检索等很多内容。
  • 文档 C：短文，只提到一次机器学习，没有提到检索。

BM25 大概率会给文档 A 较高分数，因为它短而集中，并且同时命中了两个查询词。

文档 B 虽然也命中了查询词，但因为篇幅长，相关性会被长度归一化适当压低。

文档 C 只命中了一部分查询词，所以分数通常不会太高。

这就是 BM25 的优势：它不只是看有没有命中关键词，还会判断命中的词是否重要、命中次数是否合理、文档长度是否公平。

BM25 和 TF-IDF 的区别

BM25 可以看作是 TF-IDF 的改进版本，但它比传统 TF-IDF 更适合搜索排序。

主要区别：

  • TF-IDF 通常直接使用词频，BM25 对词频做饱和处理。
  • BM25 显式考虑文档长度归一化。

在 RAG（Retrieval-Augmented Generation）系统里，BM25 常用于第一阶段召回。

典型流程是：

BM25 在 RAG 中尤其适合处理：

  • 专有名词，比如论文名、函数名、类名、接口名。
  • 编号和代码，比如 ERR_1024、user_id、HNSW。
  • 精确短语，比如配置项、命令、产品型号。
  • 用户明确输入关键词的场景。

向量检索擅长语义相似，但有时会漏掉精确字符串；BM25 正好补上这个短板。所以很多检索系统会使用 hybrid search，也就是 BM25 + dense embedding 的混合检索。

BM25 的优点

BM25 的优点非常明确：

  • 实现简单，计算速度快。
  • 不依赖训练数据。
  • 对精确关键词匹配非常强。
  • 可解释性好，可以知道哪些词贡献了分数。
  • 适合作为搜索系统的强 baseline。
  • 和向量检索互补，适合混合检索。

这也是为什么即使进入大模型时代，BM25 依然没有过时。

BM25 的局限

BM25 也有明显局限：

  • 不理解语义相似，比如“汽车”和“车辆”如果没有同时出现，就很难匹配。
  • 对分词质量敏感，中文场景尤其明显。
  • 对同义词、缩写、拼写错误不够鲁棒。
  • 只关注词面匹配，不理解上下文含义。
  • 对跨语言检索支持较弱。

所以在现代系统中，BM25 很少单独承担全部检索任务。更常见的做法是把它和向量检索、query rewrite、rerank 模型结合起来。

总结

BM25 是一个经典但仍然非常实用的关键词检索算法。它通过 IDF 衡量词的重要性，通过词频饱和避免关键词堆砌，通过文档长度归一化修正长文档优势。

它最大的价值不是“替代语义检索”，而是提供一种稳定、快速、可解释的精确匹配能力。在现代 RAG 和搜索系统中，BM25 通常和向量检索一起使用：BM25 负责抓住关键词和专有名词，向量检索负责抓住语义相似，再通过重排序模型融合两者的结果。

如果要搭建一个可靠的检索系统，BM25 依然是最值得先实现的 baseline 之一。
